## 2. 顧客獲得コスト（CAC）分析

### 2.1 CAC計算方法論

#### 2.1.1 CAC定義と計算式

**標準CAC計算式**:
```
         マーケティング費用合計 + セールス費用合計
CAC = ─────────────────────────────────────────────
              新規顧客獲得数

詳細内訳:
├── マーケティング費用:
│   ├── 広告費（デジタル・オフライン）
│   ├── コンテンツ制作費
│   ├── インフルエンサー費用
│   ├── キャンペーン費用
│   └── マーケティングツール費用
├── セールス費用:
│   ├── セールス人件費
│   ├── 営業ツール費用
│   └── 出張・交際費
└── 配賦費用:
    ├── マーケティング部門オーバーヘッド
    └── カスタマーサクセス（オンボーディング）
```

**TripTrip CAC計算式（詳細版）**:
```
                    (広告費 + コンテンツ費 + インフルエンサー費
                     + 人件費按分 + ツール費用 + 間接費配賦)
Blended CAC = ───────────────────────────────────────────────────
                            新規有料顧客数

2024年度計算例:
¥3.2億円 ÷ 128,000人 = ¥2,500/顧客
```

#### 2.1.2 CAC計算の粒度設定

**チャネル別CAC計算**:
```python
# CAC計算モデル
class CACCalculator:
    def __init__(self, fiscal_year):
        self.fiscal_year = fiscal_year
        self.channels = {
            'organic': {'cost': 0, 'acquisitions': 0},
            'paid_search': {'cost': 0, 'acquisitions': 0},
            'paid_social': {'cost': 0, 'acquisitions': 0},
            'affiliate': {'cost': 0, 'acquisitions': 0},
            'partnerships': {'cost': 0, 'acquisitions': 0}
        }

    def calculate_blended_cac(self):
        total_cost = sum(ch['cost'] for ch in self.channels.values())
        total_acq = sum(ch['acquisitions'] for ch in self.channels.values())
        return total_cost / total_acq if total_acq > 0 else 0

    def calculate_channel_cac(self, channel):
        ch = self.channels[channel]
        return ch['cost'] / ch['acquisitions'] if ch['acquisitions'] > 0 else 0

    def calculate_paid_cac(self):
        paid_channels = ['paid_search', 'paid_social', 'affiliate']
        total_cost = sum(self.channels[ch]['cost'] for ch in paid_channels)
        total_acq = sum(self.channels[ch]['acquisitions'] for ch in paid_channels)
        return total_cost / total_acq if total_acq > 0 else 0
```

**セグメント別CAC定義**:
```
顧客セグメント別CAC:
├── 新規顧客CAC: 初回購入顧客の獲得コスト
├── リアクティベーションCAC: 休眠顧客の復活コスト
├── アップセルCAC: 既存顧客のアップグレードコスト

```

## 3. 顧客生涯価値（LTV）モデル

### 3.1 LTV計算方法論

#### 3.1.1 LTV算出の基本フレームワーク

**標準LTV計算式**:
```
          ARPU × 粗利率
LTV = ─────────────────
           月次チャーン率

詳細版:
LTV = Σ(各月の期待収益 × 残存率 × 粗利率) × (1 - 割引率)^n

TripTrip適用モデル:
├── 契約型LTV: サブスクリプション収益
├── トランザクション型LTV: 都度購入収益
└── ハイブリッドLTV: 両方の合算

計算例（2027年目標）:
├── 月次ARPU: ¥6,500
├── 粗利率: 45%
├── 月次チャーン率: 3%
├── 単純LTV = ¥6,500 × 45% / 3% = ¥97,500
├── 実効LTV（割引後）: ¥32,000
└── 平均顧客寿命: 33ヶ月
```

#### 3.1.2 多層LTVモデル

**TripTrip LTV計算モデル**:
```python
class LTVCalculator:
    def __init__(self):
        self.discount_rate = 0.10  # 年率10%
        self.forecast_months = 60   # 5年予測

    def calculate_ltv(self, cohort_data):
        """
        多層LTV計算モデル
        """
        # 収益ストリーム別LTV
        ecommerce_ltv = self._calculate_stream_ltv(
            cohort_data, 'ecommerce',
            avg_order=15000, frequency=2.5, margin=0.42
        )

        service_ltv = self._calculate_stream_ltv(
            cohort_data, 'service',
            avg_order=8000, frequency=3.0, margin=0.35
        )

        rental_ltv = self._calculate_stream_ltv(
            cohort_data, 'rental',
            avg_order=12000, frequency=1.5, margin=0.65
        )

        # 合算LTV
        total_ltv = ecommerce_ltv + service_ltv + rental_ltv

        # リスク調整
        risk_adjusted_ltv = total_ltv * 0.85  # 15%リスク控除

        return {
            'ecommerce_ltv': ecommerce_ltv,
            'service_ltv': service_ltv,
            'rental_ltv': rental_ltv,
            'total_ltv': total_ltv,
            'risk_adjusted_ltv': risk_adjusted_ltv
        }

    def _calculate_stream_ltv(self, cohort, stream, avg_order, frequency, margin):
        """
        収益ストリーム別LTV計算
        """
        monthly_values = []
        for month in range(self.forecast_months):
            retention = cohort.get_retention(month)
            revenue = avg_order * frequency / 12 * retention
            profit = revenue * margin
            discount_factor = (1 + self.discount_rate/12) ** -month
            monthly_values.append(profit * discount_factor)

        return sum(monthly_values)
```

## 5. CACペイバック期間

### 5.1 計算方法と前提条件

#### 5.1.1 CACペイバック計算式

**標準CACペイバック計算**:
```
                      CAC
CACペイバック期間 = ─────────────────
                  月次貢献利益

詳細計算:
月次貢献利益 = 月次ARPU × 粗利率

TripTrip計算例（2024年）:
├── CAC: ¥2,500
├── 月次ARPU: ¥1,550（年間¥18,600 ÷ 12）
├── 粗利率: 37.5%
├── 月次貢献利益: ¥1,550 × 37.5% = ¥581
├── CACペイバック: ¥2,500 ÷ ¥581 = 4.3ヶ月
└── 実効ペイバック（リテンション調整後）: 4.8ヶ月
```

#### 5.1.2 リテンション調整ペイバック

**リテンション考慮ペイバック計算**:
```python
def calculate_payback_with_retention(cac, monthly_arpu, gross_margin, retention_curve):
    """
    リテンション調整済みCACペイバック計算
    """
    cumulative_profit = 0
    month = 0

    while cumulative_profit < cac and month < 36:
        retention_rate = retention_curve[month]
        monthly_profit = monthly_arpu * gross_margin * retention_rate
        cumulative_profit += monthly_profit
        month += 1

    return month

# TripTrip 2024年計算
cac = 2500
monthly_arpu = 1550
gross_margin = 0.375
retention_curve = [1.0, 0.68, 0.55, 0.48, 0.42, 0.38, ...]

payback_months = calculate_payback_with_retention(
    cac, monthly_arpu, gross_margin, retention_curve
)
```

## 7. ネット収益リテンション（NRR）

### 7.1 NRR計算と構成要素

#### 7.1.1 NRR算出方法

**NRR計算式**:
```
         期首MRR + 拡大MRR - 縮小MRR - チャーンMRR
NRR = ─────────────────────────────────────────────── × 100
                       期首MRR

TripTrip計算例（2024年）:
├── 期首MRR: ¥100M
├── 拡大MRR（アップセル・クロスセル）: ¥18M
├── 縮小MRR（ダウングレード）: ¥5M
├── チャーンMRR（解約）: ¥5M
├── NRR = (100 + 18 - 5 - 5) / 100 = 108%
```
