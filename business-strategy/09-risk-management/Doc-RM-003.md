# リスクマトリックスと早期警戒指標 [Doc-RM-003]

## エグゼクティブサマリー

本文書は、TripTripの全社的リスク管理体制の基盤として、財務・運用・市場・戦略の4つのカテゴリにわたる包括的なリスクマトリックスと、迅速な軌道修正を可能にする早期警戒指標システムを構築します。Goldman Sachs/JP Morganレベルの分析的厳密さで、32の主要リスク項目を特定・評価し、確率×影響度スコアリングによる優先順位付けを実施。リアルタイムモニタリングダッシュボードと、経営陣・取締役会向けの報告テンプレートを整備し、データドリブンなリスク管理を実現します。本システムの導入により、リスク顕在化の平均検知時間を2週間から3日に短縮し、対応策実施までの時間を50%削減することを目標としています。

---

## 1. はじめに & コンテキスト

### 1.1 目的 & スコープ

#### 1.1.1 文書の目的

本文書は、TripTripのリスク管理体制において、以下の目的を達成します：

**1. 包括的なリスク特定と分類**
- 全社横断的なリスクインベントリの作成
- 4カテゴリ（財務・運用・市場・戦略）での体系的分類
- リスク間の相関関係と連鎖リスクの可視化

**2. 定量的リスク評価の実施**
- 確率×影響度スコアリングフレームワークの適用
- リスクヒートマップによる優先順位の明確化
- 期待損失（Expected Loss）の定量化

**3. 早期警戒システムの構築**
- リーディング指標の定義と閾値設定
- リアルタイムモニタリングダッシュボード
- 自動アラートと迅速な対応プロセス

**4. ガバナンス体制の確立**
- 定期報告フォーマットの標準化
- 取締役会・投資家向けコミュニケーション
- 継続的改善プロセスの組み込み

#### 1.1.2 分析スコープ

```
スコープ定義:
├── 対象リスクカテゴリ:
│   ├── 財務リスク（収益、コスト、資金調達、流動性）
│   ├── 運用リスク（技術、人材、プロセス、サプライチェーン）
│   ├── 市場・競争リスク（市場環境、競合、顧客）
│   └── 戦略リスク（プロダクト、市場適合、規制）
├── 評価期間: 2024年-2027年
├── 更新頻度:
│   ├── リアルタイム: 自動モニタリング指標
│   ├── 週次: KPIダッシュボード
│   ├── 月次: リスクスコアカード
│   └── 四半期: 包括的リスクレビュー
└── 除外項目:
    ├── 個別プロジェクトリスク（PM管理）
    └── 日常業務リスク（部門管理）
```

### 1.2 方法論 & フレームワーク適用

#### 1.2.1 リスク評価フレームワーク

```
リスク評価5段階スケール:

【確率スケール】
┌─────────────────────────────────────────────────────────────┐
│ スコア │ 確率レベル   │ 定義                  │ 年間発生確率 │
├─────────────────────────────────────────────────────────────┤
│ 1     │ 非常に低い   │ ほぼ発生しない         │ <5%        │
│ 2     │ 低い        │ まれに発生             │ 5-15%      │
│ 3     │ 中程度      │ 時々発生               │ 15-40%     │
│ 4     │ 高い        │ 頻繁に発生             │ 40-70%     │
│ 5     │ 非常に高い  │ ほぼ確実に発生         │ >70%       │
└─────────────────────────────────────────────────────────────┘

【影響度スケール】
┌─────────────────────────────────────────────────────────────┐
│ スコア │ 影響レベル     │ 財務影響     │ 事業影響          │
├─────────────────────────────────────────────────────────────┤
│ 1     │ 軽微          │ <¥0.5億     │ 日常業務に軽微な影響│
│ 2     │ 小            │ ¥0.5-2億    │ 一部業務に影響     │
│ 3     │ 中            │ ¥2-5億      │ 重要業務に影響     │
│ 4     │ 大            │ ¥5-15億     │ 事業計画に重大影響 │
│ 5     │ カタストロフィック│ >¥15億    │ 事業継続に影響     │
└─────────────────────────────────────────────────────────────┘

【リスクスコア計算】
リスクスコア = 確率スコア × 影響度スコア

リスク優先度分類:
- 極高リスク（スコア20-25）: 即時対応必須
- 高リスク（スコア12-19）: 優先対応
- 中リスク（スコア6-11）: 計画的対応
- 低リスク（スコア1-5）: モニタリング継続
```

#### 1.2.2 早期警戒指標フレームワーク

```
早期警戒指標の種類:
┌─────────────────────────────────────────────────────────────┐
│ 指標タイプ      │ 定義                     │ 用途           │
├─────────────────────────────────────────────────────────────┤
│ リーディング指標│ リスク顕在化の先行指標   │ 予防的対応     │
│ ラギング指標    │ リスク顕在化後の結果指標 │ 影響度測定     │
│ 複合指標        │ 複数指標の組み合わせ     │ 総合的リスク評価│
└─────────────────────────────────────────────────────────────┘

アラートレベル定義:
├── グリーン（正常）: 閾値内で安定
├── イエロー（注意）: 閾値の80%到達
├── オレンジ（警告）: 閾値到達
└── レッド（緊急）: 閾値の120%超過
```

### 1.3 主要な前提条件 & 制約

```
前提条件:
├── データ可用性:
│   ├── 財務データ: リアルタイム〜日次
│   ├── 運用データ: リアルタイム
│   ├── 市場データ: 週次〜月次
│   └── 戦略データ: 月次〜四半期
├── リソース前提:
│   ├── リスク管理担当: CFO兼任（初期）→専任（シリーズB後）
│   ├── モニタリングツール: BI+カスタムダッシュボード
│   └── レポーティング: 自動化+手動分析
└── 制約事項:
    ├── スタートアップ特性（過去データ限定）
    ├── 外部データの正確性
    └── 予測モデルの限界
```
