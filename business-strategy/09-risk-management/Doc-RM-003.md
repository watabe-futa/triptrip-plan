# リスクマトリックスと早期警戒指標 [Doc-RM-003]

## エグゼクティブサマリー

本文書は、TripTripの全社的リスク管理体制の基盤として、財務・運用・市場・戦略・規制の5つのカテゴリにわたる包括的なリスクマトリックスと、迅速な軌道修正を可能にする早期警戒指標システムを構築します。Goldman Sachs/JP Morganレベルの分析的厳密さで、50以上の主要リスク項目を特定・評価し、確率×影響度×検出可能性によるリスク優先度数（RPN）スコアリングで優先順位付けを実施。リアルタイムモニタリングダッシュボードと、経営陣・取締役会向けの報告テンプレートを整備し、データドリブンなリスク管理を実現します。本システムの導入により、リスク顕在化の平均検知時間を2週間から3日に短縮し、対応策実施までの時間を50%削減することを目標としています。

## 1. はじめに & リスク管理フレームワーク

### 1.1 目的 & スコープ

#### 1.1.1 文書の目的

本文書は、TripTripのリスク管理体制において、以下の目的を達成します：

**1. 包括的なリスク特定と分類**
- 全社横断的なリスクインベントリの作成
- 5カテゴリ（財務・運用・市場・戦略・規制）での体系的分類
- リスク間の相関関係と連鎖リスクの可視化

**2. 定量的リスク評価の実施**
- 確率×影響度×検出可能性によるRPNスコアリング
- リスクヒートマップによる優先順位の明確化
- 期待損失（Expected Loss）の定量化

**3. 早期警戒システムの構築**
- リーディング指標の定義と閾値設定
- リアルタイムモニタリングダッシュボード
- 自動アラートと迅速な対応プロセス

**4. ガバナンス体制の確立**
- 定期報告フォーマットの標準化
- 取締役会・投資家向けコミュニケーション
- 継続的改善プロセスの組み込み

#### 1.1.2 分析スコープ

```
スコープ定義:
├── 対象リスクカテゴリ:
│   ├── 財務リスク（収益、コスト、資金調達、流動性）
│   ├── 運用リスク（技術、人材、プロセス、サプライチェーン）
│   ├── 市場・競争リスク（市場環境、競合、顧客）
│   ├── 戦略リスク（プロダクト、市場適合、成長戦略）
│   └── 規制・コンプライアンスリスク（法務、データ保護、業法）
├── 評価期間: 2024年-2028年
├── 更新頻度:
│   ├── リアルタイム: 自動モニタリング指標
│   ├── 週次: KPIダッシュボード
│   ├── 月次: リスクスコアカード
│   └── 四半期: 包括的リスクレビュー
└── 除外項目:
    ├── 個別プロジェクトリスク（PM管理）
    └── 日常業務リスク（部門管理）
```

### 1.2 リスク評価方法論

#### 1.2.1 FMEA拡張フレームワーク

```
リスク優先度数（RPN）計算:
RPN = 発生確率 × 影響度 × 検出困難度

【発生確率スケール（O: Occurrence）】
┌─────────────────────────────────────────────────────────────┐
│ スコア │ 確率レベル   │ 定義                  │ 年間発生確率 │
├─────────────────────────────────────────────────────────────┤
│ 1     │ 極めて低い   │ ほぼ発生しない         │ <1%        │
│ 2     │ 非常に低い   │ まれに発生可能性あり   │ 1-5%       │
│ 3     │ 低い        │ 時々発生する可能性     │ 5-15%      │
│ 4     │ 中程度      │ 繰り返し発生の可能性   │ 15-40%     │
│ 5     │ 高い        │ 頻繁に発生             │ 40-70%     │
│ 6     │ 非常に高い  │ ほぼ確実に発生         │ >70%       │
└─────────────────────────────────────────────────────────────┘

【影響度スケール（S: Severity）】
┌─────────────────────────────────────────────────────────────┐
│ スコア │ 影響レベル     │ 財務影響     │ 事業影響          │
├─────────────────────────────────────────────────────────────┤
│ 1     │ 軽微          │ <¥0.3億     │ 日常業務に影響なし │
│ 2     │ 小            │ ¥0.3-1億    │ 一部業務に軽微な影響│
│ 3     │ 中            │ ¥1-3億      │ 重要業務に影響     │
│ 4     │ 大            │ ¥3-8億      │ 複数部門に重大影響 │
│ 5     │ 重大          │ ¥8-15億     │ 事業計画に重大影響 │
│ 6     │ カタストロフィック│ >¥15億    │ 事業継続に影響     │
└─────────────────────────────────────────────────────────────┘

【検出困難度スケール（D: Detection）】
┌─────────────────────────────────────────────────────────────┐
│ スコア │ 検出レベル     │ 定義                              │
├─────────────────────────────────────────────────────────────┤
│ 1     │ 確実に検出     │ 自動検出・即時アラート            │
│ 2     │ 高確率で検出   │ 定期モニタリングで検出            │
│ 3     │ 中程度で検出   │ 詳細分析で検出                    │
│ 4     │ 検出困難      │ 専門的分析が必要                  │
│ 5     │ 非常に困難    │ 偶発的にしか検出できない          │
│ 6     │ 検出不能      │ 顕在化するまで検出不能            │
└─────────────────────────────────────────────────────────────┘

RPN優先度分類:
- 極高リスク（RPN 100-216）: 即時対応必須
- 高リスク（RPN 50-99）: 優先対応
- 中リスク（RPN 25-49）: 計画的対応
- 低リスク（RPN 1-24）: モニタリング継続
```

## 10. リスク定量化フレームワーク【拡充セクション】

### 10.1 Value at Risk (VaR) 計算手法

```
VaR計算手法の概要:
┌─────────────────────────────────────────────────────────────────────────────┐
│ VaR (Value at Risk) = 特定の信頼水準で、一定期間内に発生しうる最大損失額    │
│                                                                             │
│ 計算式: VaR = μ - Zα × σ                                                   │
│ ├── μ: 期待リターン（または損失）                                          │
│ ├── Zα: 信頼水準に対応する標準正規分布の分位点                             │
│ └── σ: リターンの標準偏差                                                  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 10.3 リスク調整後収益率（RAROC）

```
RAROC (Risk-Adjusted Return on Capital) フレームワーク:
┌─────────────────────────────────────────────────────────────────────────────┐
│ RAROC = (収益 - 期待損失 - 営業コスト) / 経済資本                           │
│                                                                             │
│ 構成要素:                                                                   │
│ ├── 収益: 各事業セグメントの売上総利益                                     │
│ ├── 期待損失: 各リスクカテゴリの期待損失額                                 │
│ ├── 営業コスト: 配賦された営業費用                                         │
│ └── 経済資本: VaR(99%)ベースの必要資本                                     │
└─────────────────────────────────────────────────────────────────────────────┘

事業セグメント別RAROC分析（2027年予測）:
┌─────────────────────────────────────────────────────────────────────────────┐
│ セグメント     │ 収益   │ 期待損失│ 営業コスト│ 経済資本│ RAROC  │ 評価   │
├─────────────────────────────────────────────────────────────────────────────┤
│ Eコマース     │ ¥25.3億│ ¥2.5億 │ ¥15.2億  │ ¥22.0億│ 34.5%  │ 良好   │
│ サービス予約  │ ¥19.7億│ ¥1.8億 │ ¥12.5億  │ ¥18.5億│ 29.2%  │ 良好   │
│ レンタル      │ ¥11.3億│ ¥0.8億 │ ¥7.8億   │ ¥12.0億│ 22.5%  │ 許容   │
├─────────────────────────────────────────────────────────────────────────────┤
│ 全社          │ ¥56.3億│ ¥5.1億 │ ¥35.5億  │ ¥52.5億│ 29.9%  │ 良好   │
└─────────────────────────────────────────────────────────────────────────────┘

RAROC評価基準:
├── 35%以上: 優秀（リソース追加投入推奨）
├── 25-35%: 良好（現状維持）
├── 15-25%: 許容（効率改善検討）
├── 10-15%: 要改善（構造見直し必要）
└── 10%未満: 不可（撤退検討）

経済資本配分への示唆:
├── Eコマース: RAROC最高 → 追加投資優先
├── サービス予約: 安定的 → 規模拡大推奨
└── レンタル: RAROC最低 → 効率改善施策実施
```
