# TripTrip価格戦略・アーキテクチャ [Doc-BM-004]

## エグゼクティブサマリー

本文書は、TripTripの価格戦略とプライシングアーキテクチャを体系的に設計し、顧客価値に基づく最適な価格設定戦略を定義します。Doc-BM-002で特定した顧客セグメントの支払意思と、Doc-BM-003で設計した収益モデルを統合し、競争力と収益性を両立する価格体系を構築します。Hermann Simon/McKinseyレベルのプライシング戦略品質で、行動経済学に基づく科学的アプローチと実装可能な価格テスト戦略を提示します。

### 主要ハイライト
- **価値ベース価格設定**: 顧客が認識する価値に基づく価格導出（総経済的価値96,750円/旅行）
- **4段階のティアアーキテクチャ**: 無料、プレミアム、VIP、エンタープライズ
- **ダイナミックプライシング**: AI活用による需要連動型価格最適化（季節変動係数±40%）
- **地域別価格戦略**: 購買力パリティを考慮した市場別価格設定（8地域対応）
- **行動経済学応用**: アンカリング、デコイ効果、端数価格心理学の統合設計
- **サービス別最適化**: ホテル（10-15%）、チケット（8-15%）、着物（50-60%粗利）の差別化設計

## 第1章 はじめに & コンテキスト

### 1.1 目的 & スコープ

#### 1.1.1 文書の目的

本文書は、TripTripの価格戦略を設計し、以下の目的を達成することを目指します：

1. **価値ベース価格設定**: 顧客価値に基づく価格の正当化
2. **競争力のあるポジショニング**: 市場での適切な価格位置付け
3. **収益最大化**: 価格弾力性を活用した最適価格の発見
4. **顧客セグメント別最適化**: セグメントに応じた価格差別化
5. **行動経済学の応用**: 科学的知見に基づく価格設計

#### 1.1.2 プライシング戦略の重要性

```
プライシングが収益に与えるインパクト:
├─ 価格1%向上の効果: 営業利益+11%（McKinsey調査）
├─ 価格10%下落の補填: 販売数量+30%必要
├─ 価格最適化による収益改善: 平均+25%
└─ 不適切な価格設定の機会損失: 年間収益の15-20%
```

TripTripにおけるプライシングの戦略的重要性：
- **複合収益モデル**: 取引手数料、サブスク、広告の最適ミックス
- **多様なサービス**: ホテル、チケット、着物レンタル、Eコマースの統合価格設計
- **グローバル展開**: 地域別購買力に応じた価格差別化
- **競争環境**: OTA、現地事業者との価格競争力確保

### 1.2 価格戦略方法論

**採用フレームワーク**:
- 価値ベース価格設定（Value-Based Pricing）
- Van Westendorp価格感度分析

## 第2章 プライシング理論的基盤

### 2.1 価格戦略の3つのアプローチ

#### 2.1.1 価値ベースプライシング（Value-Based Pricing）

```
価値ベース価格設定の原則:

顧客価値 = 経済的価値 + 心理的価値 + ブランド価値
        ↓
最適価格 = 顧客価値 × 価値取得率（10-20%）

TripTrip適用:
├─ 時間節約価値: 42,750円（9時間削減 × 3,000円 + 待ち時間削減）
├─ コスト削減価値: 21,000円（8%割引 + 隠れコスト回避）
├─ 体験向上価値: 33,000円（発見体験 + 文化アクセス + 安心感）
├─ 総経済的価値: 96,750円/旅行
└─ 価値取得率15%: 推奨価格14,513円/旅行
```

**価値ベース価格設定のメリット**:
1. 顧客の支払意思に基づく科学的根拠
2. 価格交渉における論理的正当化
3. プレミアムポジションの確立
4. 収益最大化と顧客満足の両立

**価値ベース価格設定の実装ステップ**:
```
Step 1: 顧客価値の定量化
├─ 時間節約の金銭換算
├─ コスト削減効果の測定
├─ 体験価値の推定
└─ 代替手段との比較

Step 2: 価値取得率の決定
├─ 業界標準: 10-20%
├─ 競争環境に応じた調整
└─ 顧客セグメント別最適化

Step 3: 価格レンジの設定
├─ 保守的: 10%取得
├─ 中央値: 15%取得
└─ 積極的: 20%取得

Step 4: 市場テストと検証
├─ A/Bテスト実施
├─ 価格弾力性測定
└─ フィードバック収集
```

#### 2.1.2 競争ベースプライシング（Competitive Pricing）

```
競合価格ベンチマーク（テイクレート）:

プラットフォーム │ ホテル │ 体験  │ チケット │ 戦略
─────────────────┼────────┼───────┼──────────┼─────────
Booking.com     │ 15-18% │ -     │ -        │ プレミアム
Expedia         │ 18-25% │ 15-20%│ -        │ 高価格
Klook           │ -      │ 20-25%│ 15-20%   │ 中価格
Viator          │ -      │ 20-30%│ -        │ 高価格
GetYourGuide    │ -      │ 20-25%│ 15-20%   │ 中価格
じゃらん        │ 8-12%  │ 10-15%│ 8-12%    │ 低価格
─────────────────┼────────┼───────┼──────────┼─────────
TripTrip目標    │ 10-12% │ 15-20%│ 10-15%   │ 競争価格+価値付加
```

**競争ベース価格戦略の類型**:
1. **価格リーダー戦略**: 業界最低価格を維持
2. **価格フォロワー戦略**: 市場リーダーに追随
3. **差別化価格戦略**: 付加価値による価格プレミアム
4. **浸透価格戦略**: 低価格で市場シェア獲得

**TripTripの競争価格ポジション**:
```
ポジショニング: 差別化×競争価格ハイブリッド

├─ ホテル予約: 競争価格（じゃらん水準）で市場浸透
│   └─ 理由: 高弾力性カテゴリ、ボリューム重視
├─ アクティビティ: 中価格（Klook水準）で価値訴求
│   └─ 理由: 中弾力性、差別化可能
├─ 着物レンタル: プレミアム価格で独自性確立
│   └─ 理由: 低弾力性、専門性による参入障壁
└─ チケット: 競争価格で利便性訴求
    └─ 理由: 高弾力性、在庫確保が重要
```

#### 2.1.3 コストプラスプライシング（Cost-Plus Pricing）

```
コストプラス価格設定:

価格 = 変動費 + 固定費配賦 + 目標利益マージン

サービス別コスト構造:
┌─────────────────────────────────────────────────────────────┐
│サービス     │変動費率│固定費配賦│目標マージン│推奨テイクレート│
├─────────────┼────────┼──────────┼────────────┼────────────────┤
│ホテル予約   │2%      │3%        │5-7%       │10-12%          │
│チケット販売 │1%      │2%        │5-7%       │8-12%           │
│アクティビティ│3%      │4%        │8-10%      │15-20%          │
│着物レンタル │15%     │10%       │25-35%     │50-60%（粗利）  │
│商品販売     │55%     │5%        │5-10%      │35-45%（粗利）  │
└─────────────┴────────┴──────────┴────────────┴────────────────┘
```

**コストプラス価格設定の留意点**:
- メリット: 確実な利益確保、計算の容易さ
- デメリット: 顧客価値や競争環境を反映しない
- TripTrip適用: 下限価格（フロア）の設定に活用

### 7.2 価格弾力性測定

```
価格弾力性測定方法論:

測定アプローチ:
├─ A/Bテスト: 直接的な価格変更による反応測定
├─ コンジョイント分析: 属性別支払意思の分解
├─ 履歴データ分析: 過去の価格変更と需要の相関
└─ 競合価格連動: 競合価格変動時の自社需要変化

カテゴリ別価格弾力性:
┌─────────────────────────────────────────────────────────────┐
│カテゴリ        │弾力性  │解釈              │価格戦略        │
├────────────────┼────────┼──────────────────┼────────────────┤
│ホテル予約      │-1.5    │高弾力            │競争価格重視    │
│アクティビティ  │-0.9    │中弾力            │価値訴求可能    │
│着物レンタル    │-0.6    │低弾力            │プレミアム可能  │
│チケット        │-1.8    │高弾力            │低マージン戦略  │
│サブスクリプション│-0.7   │低弾力            │価値ベース価格  │
│商品販売        │-1.3    │中〜高弾力        │競争+バンドル   │
└────────────────┴────────┴──────────────────┴────────────────┘

弾力性の解釈:
├─ |ε| < 1: 非弾力的 → 価格上げても需要維持、収益増
├─ |ε| = 1: 単位弾力的 → 価格変更と収益変化が相殺
└─ |ε| > 1: 弾力的 → 価格下げでボリューム増、収益増可能

最適価格計算:
P* = MC × ε / (ε + 1)
例: MC=¥1,000、ε=-2.0 → P* = ¥1,000 × 2.0 / 1.0 = ¥2,000
```

```
用語定義:

├─ ARPU（Average Revenue Per User）: ユーザー1人あたり平均収益

├─ GMV（Gross Merchandise Value）: 総流通額
├─ 粗利益率: （売上 - 売上原価）/ 売上
└─ 貢献利益: 売上 - 変動費

├─ 価格弾力性: 価格変化に対する需要変化の感度
├─ 非弾力的（|ε| < 1）: 価格変更に需要が鈍感
├─ 弾力的（|ε| > 1）: 価格変更に需要が敏感
└─ 交差弾力性: 競合価格変動に対する自社需要変化

行動経済学用語:
├─ アンカリング効果: 最初の情報が判断基準になる現象

└─ バンドル効果: セット販売による価値認識向上

ダイナミックプライシング用語:
├─ サージプライシング: 需要急増時の価格上昇
├─ イールドマネジメント: 収益最大化のための価格調整
├─ 季節変動係数: 季節による需要変動の指標
```

## 付録C: 価格シミュレーションモデル

```
収益シミュレーションテンプレート:

入力変数:
├─ 基本価格: [入力]
├─ 予想転換率: [入力]
├─ 想定トラフィック: [入力]
├─ 変動費率: [入力]
└─ 価格弾力性: [入力]

計算式:
├─ 売上 = 価格 × 転換率 × トラフィック
├─ 粗利 = 売上 × (1 - 変動費率)
├─ 価格変更時売上 = 新価格 × 新転換率 × トラフィック
│   └─ 新転換率 = 転換率 × (1 + 弾力性 × 価格変化率)
└─ 収益インパクト = 新売上 - 旧売上

シミュレーション例（ホテル予約手数料）:
├─ 現状: 手数料率10%、GMV 5億円/月、転換率2.5%
├─ 変更案: 手数料率12%（+20%）
├─ 弾力性: -1.5
├─ 新転換率: 2.5% × (1 + (-1.5) × 0.20) = 1.75%
├─ 新GMV: 5億円 × (1.75/2.5) = 3.5億円
├─ 旧収益: 5億円 × 10% = 5,000万円
├─ 新収益: 3.5億円 × 12% = 4,200万円
└─ 結論: 収益-16%、値上げ非推奨
```
